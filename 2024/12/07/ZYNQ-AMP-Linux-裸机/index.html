ZYNQ AMP Linux + 裸机

1. 硬件准备

  • Xilinx ZYNQ7020 SoC开发板 : 笔者这里使用的是微相的ZYNQ7020开发板,当然只要是ZYNQ7020的开发板都可以
  • 串口线: 用于连接开发板和电脑，登录Linux系统和监视cpu1的输出
  • JTAG线:用于调试裸机程序

2. 软件准备

  • Vivado 2020.2
  • Vitis 2020.2
  • petalinux 2020.2
  • ubuntu 18.04

vivado 和 vitis的安装在网上一搜一大把这里就不多说了，petalinux的安装博主另开一篇博客再讲

开发要求

如果你使用的是XC7Z010芯片，那么就不用再往下看了，因为XC7Z010只有1个核心，因此是没办法使用AMP的，如果你既想使用Linux，又想使用裸机，可以考虑在PL端使用microblaze,利用软核来实现实时控制。
博主这里使用XC7Z020，这个芯片拥有双核A9，因此可以一个核使用Linux，另一个核使用裸机。

博主目前有一个需求是，使用一个核心来响应外设的中断，完成实时的控制，用另一个核心来运行Linux，主要是在USB端接了一个USB网卡，裸机实现基本没可能，所以只能用Linux，但是Linux又不能实时响应中断，因此就有了这篇博客。

3. 实现思路

CPU0负责运行Linux系统，CPU1负责运行裸机控制程序。

3.1 vivado构建工程

petalinux需要vivado导出的xsa硬件工程来生成设备树和驱动程序，因此需要首先在vivado中构建硬件工程。

构建工程可以参考正点原子vitis开发教程，配置好DDR，UART,USB等外设，然后生成bitstream文件，最后导出硬件工程，工程文件即xsa后缀的文件。

3.2 构建fsbl

ZYNQ启动时，会首先运行一段BOOTROM程序，这是出厂就写死的，BOOTROM程序在完成一些芯片初始化工作之后会启动FSBL程序，FSBL程序会初始化DDR，UART等外设，然后根据启动引脚配置，选择启动方式，包括从JTAG启动，从SD卡启动，从FLASH启动等。本博客选择从SD卡启动，FSBL会从SD卡的BOOT分区读取BOOT.BIN文件，然后读取打包进BOOT.BIN的文件，将这些文件依次写入对应的地址，然后加载并运行uboot程序，uboot程序会加载并运行Linux内核。
因此我们需要构建一个FSBL程序，来加载CPU1的程序，并启动CPU1。

如何启动CPU1请参考ug585 chapter6，这里就不再赘述了，这里主要讲一下如何构建FSBL程序。

FSBL程序需要使用Vitis来构建，Vitis的安装可以参考正点原子vitis开发教程，构建过程如下：

  1. 打开Vitis，根据先前从vivado导出的xsa文件创建平台，创建过程参考正点原子vitis开发教程。
  2. 创建应用工程，模板选择zynq FSBL
  3. 在main.c中添加CPU1启动代码，参考ug585 chapter6

3.4 CPU0运行Linux

在使用amp之前请一定首先要自己尝试配置好一个可以在芯片上正常运行的Linux系统，因为amp的配置和Linux的配置是差不多的，以下就默认你知道如何配置一个可以正常运行的Linux系统。

CPU0使用的linux系统通过petalinux来制作，将vivado导出的xsa后缀工程文件拷贝到Ubuntu系统下，构建过程参考ug1144，petalinux命令查询可以参考ug1157.

-n 指定工程名，–template zynq指定使用zynq模板，–get-hw-description 指定硬件工程文件。

运行petalinux-config 进入配置界面
进入DTG Settings -> kernel Bootargs -> 取消选中generate boot args automatically,然后手动添加console=ttyPS0,115200 maxcpus=1 clk_ignore_unused root=/dev/mmcblk0p2 rw earlyprintk rootwait，然后保存退出。
maxcpus=1表示只使用一个核心,防止linux同时使用两个核心，抢占CPU1裸机程序的核心
clk_ignore_unused表示忽略未使用的时钟，Linux的电源管理策略会默认关闭未使用的时钟，导致CPU1使用的部分外设时钟被关闭，无法运行，
root=/dev/mmcblk0p2表示根文件系统在mmcblk0p2分区，
rw表示可读可写，
earlyprintk表示在内核启动时打印信息，
rootwait表示等待根文件系统挂载。
